# Windows 端口进程终止工具 (pkill)

## 概述

这是一个Windows命令行工具，用于简化通过端口号查找并终止进程的操作。它封装了`netstat`和`taskkill`命令，提供了交互式界面，让用户可以方便地管理占用特定端口的进程。

## 使用方法

### 基本用法
```
pkill [端口号] [taskkill参数]
```

### 示例
```
# 终止占用8080端口的进程（默认强制终止）
pkill 8080

# 强制终止占用3000端口的进程及其子进程
pkill 3000 /t

# 终止占用5432端口的进程，并显示详细信息
pkill 5432 /v

# 标准强制终止（与默认行为相同）
pill 3000 /f
```

### 示例输出 - 单个进程
```
找到端口 8080 对应的进程:
TCP    0.0.0.0:8080    0.0.0.0:0    LISTENING    12345

是否终止此进程? (yes/y 确认, 其他取消): y

正在终止进程...
SUCCESS: The process with PID 12345 has been terminated.
进程已成功终止
```

### 示例输出 - 多个进程
```
找到 2 个端口 3000 对应的进程:

[1] TCP    0.0.0.0:3000    0.0.0.0:0    LISTENING    54321
[2] TCP    127.0.0.1:3000  127.0.0.1:45678 ESTABLISHED  98765

请选择要终止的进程 (1-2): 1

选中的进程:
TCP    0.0.0.0:3000    0.0.0.0:0    LISTENING    54321

是否终止此进程? (yes/y 确认, 其他取消): y

正在终止进程...
进程已成功终止
```

### 参数说明
- **第一个参数**: 端口号（必需）
- **其他参数**: 直接传递给`taskkill`命令：
  - `/f` - 强制终止（如果未提供其他参数，默认添加此参数）
  - `/t` - 终止指定进程及其所有子进程
  - `/v` - 显示详细的终止操作信息
  - 更多选项请参考：`taskkill /?`

## 工作流程

1. 在命令提示符中输入`pkill [端口号]`
2. 工具搜索并显示所有匹配的进程，包含完整的网络连接信息：
   - **单个结果**: 自动显示详细连接信息
   - **多个结果**: 显示带序号的选项供用户选择
3. **对于多个结果**: 用户通过输入对应编号选择进程
4. **确认提示**: "是否终止此进程? (yes/y 确认, 其他取消)"
5. 如果确认，工具执行终止操作并显示结果

### 默认行为
- **强制终止 (`/f`)**: 如果未提供额外参数，将自动添加
- **用户指定参数**: 优先级高于默认值
- **智能选择**: 自动适应单个或多个进程场景

## 安装方法

### 方法一：添加到系统PATH（推荐）

1. 复制`pkill.bat`到固定位置，例如：`C:\Tools\`
2. 右键点击"此电脑"或"我的电脑"，选择"属性"
3. 点击"高级系统设置"
4. 点击"环境变量"
5. 在"系统变量"部分，找到并选择"Path"变量，点击"编辑"
6. 点击"新建"，然后输入存放`pkill.bat`的文件夹路径（例如：`C:\Tools\`）
7. 点击"确定"保存所有更改
8. 重启所有打开的命令提示符窗口

### 方法二：复制到系统目录

1. 复制`pkill.bat`到`C:\Windows\System32\`目录（需要管理员权限）

## 技术细节

- 使用`C:\Windows\System32\netstat.exe`查找进程
- 使用完整系统路径确保兼容性
- 使用代码页936 (GBK)支持中文显示
- 健壯的参数传递处理，避免常见的批处理脚本问题
- 显示完整的网络连接信息：协议、本地地址、远程地址、连接状态、PID
- 支持智能单进程/多进程选择

## 注意事项

- 终止系统进程可能需要管理员权限
- 强制终止进程可能导致数据丢失，请谨慎使用
- 如果同一端口被多个进程使用，只会终止选中的进程
- 该工具仅显示完整的IP地址和端口信息，避免编码问题
- 工具会在需要时自动设置强制终止参数
- 通过序号选择确保用户可以精确控制要终止的进程
